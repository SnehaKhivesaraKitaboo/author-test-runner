Kitaboo Authoring Test Runner

Team demo summary — what it is, how it works, and how to present it to stakeholders.

Folder: closify/authoring-tests · June 2026

What is it?

authoring-tests is a standalone automation suite + web dashboard for the Kitaboo Authoring Tool. It runs Cypress tests against a real staging authoring URL, shows live progress and screenshots, and organizes tests by module (ELA, World Languages, Math) and component category (Activity Creation, Assessment Widgets, Layouts, and more).

One-line pitch: Select module → select component → run → watch live → download report.

Why we built it

  • Manual QC for dozens of authoring components is slow and inconsistent.
  • QC test cases live in PDFs — we mapped them in a component registry.
  • Each run is isolated: logs, HTML/PDF report, screenshots, and video per run.
  • Users sign in; run history is stored in MySQL.

How it works (step by step)

  1. Start the server: npm start in authoring-tests → open http://localhost:4321
  2. Log in (demo user or your account).
  3. On the dashboard, paste a launch URL (staging link with courseId and userToken).
  4. Choose an Authoring Module: ELA, WL, or Math.
  5. Pick Test Mode: Full E2E (all spec files) or Component Test (selected widgets).
  6. Use the category dropdown to filter components (e.g. Assessment Widgets).
  7. Click components to select them; for one deep component, the 8-step test flow appears.
  8. Click Run Component Tests (or Run E2E Suite).
  9. Cypress automates the real authoring app; the dashboard streams logs and progress.
  10. Right panel: live preview — flow checklist + screenshots during the run.
  11. When finished: pass/fail summary, reports, video, and run history.

5-minute demo script

  1. Problem — “We need repeatable QC on authoring widgets, not only manual testing.”
  2. Solution — “Node server + Cypress + dashboard in authoring-tests.”
  3. Modules — “Tests mirror the product: ELA / WL / Math and categories like Assessment Widgets.”
  4. Live run — ELA → Assessment Widgets → Fill in the Blank → Run → show progress + preview.
  5. Status — FIB deep is most complete; others are basic smoke or planned from QC PDFs.

Demo tips

  • Use a fresh launch URL — userToken expires quickly.
  • For FIB deep QC, select only “Fill in the Blank”.
  • Hard refresh the dashboard (Ctrl+Shift+R) if a selected card looks hidden.
  • Runs use headless Chrome; video shows the authoring app only.
