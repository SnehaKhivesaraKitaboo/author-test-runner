1. Executive Summary

The Authoring Automation Framework (authoring-tests) is already a functional local test runner with a custom dashboard, SSE live logs, PDF reports, and component-wise testing. Compared to kitaboo-auto-tester, it leads on authoring-specific depth (12 specs, live preview, component registry) but lags on DevOps, stability patterns, and enterprise reporting/analytics.

Strengths to Preserve

  • Express orchestrator + per-run artifact isolation (runs/run-<id>/)
  • Custom HTML/PDF pipeline (cypress.config.ts + Puppeteer)
  • Live dashboard (SSE, screenshot preview, component picker)
  • Deep authoring coverage (specs 09–12, 47 components)
  • Angular scope injection pattern (reliable for drag-drop)

Critical Gaps

  • No Docker, CI, notifications, or Allure
  • No Page Object Model / centralized locators
  • testIsolation: false, heavy cy.wait(), duplicated registries
  • Hardcoded userToken in UI + fixtures
  • In-memory history (lost on server restart)
  • test-registry.json (3,231 manual TCs) is never wired to code

5. Phased Migration Plan

Phase 0 — Security & Stability (Week 1)

  • Remove hardcoded tokens → .env + dashboard input only
  • Add Cypress retries: retries: { runMode: 2 }
  • Fix exception handler — fail on unknown errors
  • Accurate progress — port kitaboo progress parser
  • Persistent history — scan runs/ on server start

Phase 1 — Architecture Foundation (Week 2–3)

  • Single registry — test-registry.json drives specs 06–09 + /api/registry
  • Extract lib/ modules from server.js (no API changes)
  • Create pages/ — EditorPage, AddPanelPage (commands delegate to pages)
  • Smart waits utility — replace top 20 cy.wait() calls
  • Docker — adapt kitaboo Dockerfile (port 4321)
  • GitHub Actions — CI workflow + artifact upload

Phase 2 — Reporting & Dashboard (Week 4–5)

  • Embed screenshots/videos in HTML/PDF reports
  • Failure analysis section (stack trace, browser, env)
  • Dashboard: Chart.js trends, module filter, dark/light mode
  • /api/test-cases/:spec for MCQ deep TC picker
  • Allure plugin (optional parallel reporter)

Phase 3 — Enterprise Scale (Week 6–8)

  • CI matrix sharding (layouts / elements / widgets / deep)
  • Slack webhook on failure
  • Auto-healing locator chains for top 10 flaky selectors
  • testIsolation: true rollout spec-by-spec
  • Analytics API + execution timeline
  • ESLint + strict TypeScript (incremental)
